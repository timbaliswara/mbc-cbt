1. Ringkasan Aplikasi

MBC CBT Online adalah aplikasi tes berbasis Laravel Livewire untuk mengelola ujian online SD dan SMP. Sistem memakai pembayaran manual: siswa menghubungi admin, admin memverifikasi pembayaran, lalu admin memberikan token ujian satu kali pakai.

Tipe soal MVP yang didukung:

Pilihan ganda teks Pilihan ganda dengan gambar soal Pilihan ganda dengan gambar opsi Esai teks Esai dengan gambar Stimulus bacaan/gambar

2. Alur Besar Sistem

  1. Admin login ke dashboard.
  2. Admin membuat paket ujian dan mengatur jadwal, durasi, status, serta aturan tampil hasil.
  3. Admin menambahkan stimulus jika satu bacaan/gambar dipakai untuk beberapa soal.
  4. Admin membuat soal pilihan ganda atau esai, termasuk gambar pada pertanyaan atau opsi jawaban.
  5. Siswa melakukan pembayaran manual kepada admin.
  6. Admin membuat token ujian dan memberikannya kepada siswa.
  7. Siswa membuka portal ujian, memasukkan token, dan mengisi data diri.
  8. Siswa mengerjakan ujian. Jawaban tersimpan otomatis dan ujian auto-submit saat waktu habis.
  9. Sistem menghitung nilai pilihan ganda secara otomatis.
  10. Admin menilai esai secara manual, lalu melihat rekap nilai dan detail jawaban.

3. Panduan Admin

3.1 Login Admin

  1. Buka halaman /admin/login.
  2. Masukkan email dan password admin.
  3. Setelah berhasil, admin diarahkan ke dashboard.

3.2 Dashboard

Dashboard menampilkan ringkasan operasional:

  • Jumlah paket ujian.
  • Jumlah soal aktif.
  • Jumlah token dibuat.
  • Jumlah sesi siswa.
  • Jumlah esai yang menunggu koreksi.
  • Aktivitas ujian terbaru.

3.3 Membuat Paket Ujian

  1. Buka menu Paket Ujian.
  2. Isi nama ujian, jenjang, kelas, mata pelajaran, durasi, jadwal mulai, dan jadwal selesai.
  3. Pilih status Aktif agar siswa bisa memakai token.
  4. Aktifkan Tampilkan hasil ke siswa jika nilai boleh langsung terlihat setelah ujian.
  5. Klik Simpan paket.

3.4 Membuat Stimulus

Stimulus digunakan untuk bacaan, data, grafik, tabel, atau gambar yang dipakai beberapa soal sekaligus.

  1. Buka menu Soal.
  2. Pilih paket ujian aktif untuk input soal.
  3. Isi judul stimulus, tipe stimulus, dan konten bacaan atau upload gambar.
  4. Klik Simpan stimulus.

3.5 Membuat Soal

  1. Buka menu Soal.
  2. Pilih nomor soal dan tipe soal: pilihan ganda atau esai.
  3. Pilih stimulus jika soal memakai bacaan/gambar yang sama.
  4. Tulis pertanyaan dan upload gambar soal jika dibutuhkan.
  5. Untuk pilihan ganda, isi opsi A sampai E dan pilih kunci jawaban.
  6. Upload gambar pada opsi jika jawaban berupa gambar.
  7. Isi bobot nilai dan pembahasan opsional.
  8. Klik Simpan soal.

3.6 Membuat Token

  1. Buka menu Token.
  2. Pilih paket ujian.
  3. Isi jumlah token yang ingin dibuat.
  4. Isi waktu kedaluwarsa jika diperlukan.
  5. Klik Buat token.
  6. Berikan token kepada siswa yang sudah membayar.

Catatan: token hanya bisa dipakai satu kali. Setelah siswa mulai ujian, token berubah menjadi in progress. Setelah submit, token berubah menjadi used.

3.7 Melihat Rekap Nilai

Buka menu Hasil. Admin dapat melihat:

  • Total nilai.
  • Nilai pilihan ganda.
  • Nilai esai.
  • Jumlah benar, salah, dan kosong.
  • Durasi pengerjaan.
  • Waktu mulai dan selesai.
  • Status lulus atau belum lulus.
  • Detail jawaban per soal.

3.8 Menilai Esai

  1. Buka menu Hasil.
  2. Klik Lihat detail pada hasil siswa.
  3. Cari baris soal bertipe esai.
  4. Isi skor esai pada kolom skor.
  5. Klik Simpan.
  6. Total nilai akan diperbarui berdasarkan nilai pilihan ganda + nilai esai.

4. Panduan Siswa

4.1 Masuk Ujian

  1. Buka halaman /ujian.
  2. Masukkan token dari admin.
  3. Isi nama lengkap, kelas, nomor HP, dan asal sekolah.
  4. Klik Mulai ujian.

4.2 Mengerjakan Ujian

  • Timer muncul di bagian atas ruang ujian.
  • Siswa dapat berpindah soal menggunakan tombol sebelumnya, berikutnya, atau navigasi nomor soal.
  • Jawaban pilihan ganda tersimpan otomatis saat dipilih.
  • Jawaban esai tersimpan otomatis saat diketik.
  • Tombol ragu-ragu dapat dipakai untuk menandai soal.
  • Jika waktu habis, sistem melakukan auto-submit.

4.3 Selesai Ujian

  1. Klik Submit jika sudah selesai.
  2. Konfirmasi submit ujian.
  3. Jika admin mengaktifkan hasil siswa, nilai akan tampil setelah submit.
  4. Jika ada esai, nilai akhir menunggu koreksi admin.

6. Tips Operasional

  • Pastikan status paket ujian adalah Aktif sebelum token diberikan ke siswa.
  • Gunakan stimulus untuk bacaan panjang agar soal lebih mudah dikelola.
  • Gunakan token berbeda untuk setiap siswa agar riwayat pengerjaan tidak bercampur.
  • Periksa menu Hasil setelah ujian selesai, terutama jika ada soal esai.
  • Jangan memberikan ulang token yang statusnya sudah used atau in progress.
